[笔记]IM相关知识

本文是对于《iOS即时通讯，从入门到“放弃”？》涂耀辉 中即时通讯相关知识的总结，方便自己理解，如需更多了解，请看原文，如有侵权，请告知

传输协议

  • TCP
  • UDP

聊天协议

  • Socket 原生
  • WebScoket
  • MQTT
  • XMPP

MQTT 和 XMPP 为聊天协议 是最上层的协议

WebSocket 是传输通讯协议，它是基于 Socket 封装的一个协议

通常所说的IM私有协议，就是基于 WebScoket 或者 Socket 原生进行分装的一个聊天协议

简单的 IM 客户端需要做的事情：

  1. 客户端调用 Socket, 创建 socket
  2. 客户端调用 connect, 向服务器发起请求以建立连接
  3. 客户端与服务器建立连接后，通过 send/receive 向服务器发送数据或者从服务器接受收据
  4. 客户端调用 close 关闭 socket

服务器所需要做的工作：

  1. 服务器调用 socket 创建 socket
  2. 服务器调用 listen 设置缓冲区
  3. 服务器通过 accept 接受客户端请求并建立连接
  4. 服务器与客户端建立连接后，就可以通过 send/receive 向客户端发送或者从客户端接受数据
  5. 服务器调用 close 关闭 socket

心跳

心跳就是用来检测TCP 连接的双方是否可用
TCP本身的 keepAlive机制只能保证连接的存在，并不能保证客户以及服务端的可用性

  • 客户端发起心跳 ping 如果一段时间内没有回调，这时候主动断开连接
  • 服务器也会维护一个socket心跳间隔，一定时间内没有收到客户端发来的心跳，就主动断开连接

NAT 超时

国内运行商一般NAT超时时间为5分钟，所以心跳设置时间间隔为3-5分钟

PingPong 机制

解决心跳间隔时间间，为了保证消息的及时性，而采用双向的 PingPong 机制

服务器端发送一个 Ping ，客户端没有在约定的时间内返回响应的 ack, 则认为客户端的不在线，服务器端会主动断开socket 连接，并改为 APNs 推送的方式发送消息， 移动端同理

重连机制

理论上， 我们自己主动断开 Socket 连接，不需要重连，其他的连接断开，我们都需要进行断线重连

一般解决方案是尝试重连几次，如果无法重连成功，就不在重连

传输格式

  • ProtocolBuffer
  • JSON
  • XML
